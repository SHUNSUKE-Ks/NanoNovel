# クエストシナリオ製作依頼書

## 概要

本依頼書は、Novel PWA のクエストシナリオ（JSON形式）を外部ライターに発注する際の仕様書です。

---

## 納品物

| 項目 | ファイル名 | 形式 |
|-----|-----------|------|
| クエストシナリオ | `quest_XXX.json` | JSON |

> **XXX** はクエスト番号（3桁ゼロパディング）: `001`, `002`, `003`...

---

## ファイル配置先

```
src/assets/data/scenarios/
  └── quest_XXX.json
```

---

## storyID 命名規則

```
Q_XX_CH_TXT

Q   = Quest（固定）
XX  = クエスト番号（2桁）
CH  = チャプター番号（2桁）
TXT = テキスト連番（2〜4桁）
```

**例:**
```
Q_03_01_01  → Quest 03, Chapter 1, Text 1
Q_03_01_10  → Quest 03, Chapter 1, Text 10
Q_03_02_01  → Quest 03, Chapter 2, Text 1
```

---

## 推奨ノード数

| 項目 | 推奨値 | 上限 |
|-----|--------|------|
| 1クエストあたりのノード数 | 30〜100 | 200 |
| 1チャプターあたりのノード数 | 10〜30 | 50 |

---

## クエスト構成テンプレート

```
1. SCENE_START        ← タイトルコール表示
2. ナレーション        ← 状況説明
3. 会話（複数ノード）   ← 登場人物との会話
4. BATTLE_START       ← バトル突入（任意）
5. 会話（複数ノード）   ← バトル後の展開
6. ナレーション        ← 結果・報酬説明
7. END                ← クエスト終了
```

---

## 必須項目チェックリスト

- [ ] 全ノードに一意の `storyID` がある
- [ ] 最初のノードが `type: "SCENE_START"` である
- [ ] `questTitle` にクエスト名が設定されている
- [ ] 最後のノードに `event.type: "END"` がある
- [ ] `goto: "COLLECTION"` でクエスト一覧に戻る設定

---

## 納品時の注意事項

1. **JSONバリデーション**: 納品前に [JSONLint](https://jsonlint.com/) でエラーがないか確認
2. **文字コード**: UTF-8（BOMなし）
3. **インデント**: 4スペース推奨
4. **改行コード**: LF 推奨（CRLFでも可）
